[PTA] 基础题目集 5-37

发表于 2017-02-01 | 分类于 Programming | 阅读次数

题目要求

将一个正整数N分解成几个正整数相加，可以有多种分解方法，例如7=6+1，7=5+2，7=5+1+1，…。编程求出正整数N的所有整数分解式子。

输入格式

每个输入包含一个测试用例，即正整数$N(0<N≤30)$。

输出格式

按递增顺序输出$N$的所有整数分解式子。递增顺序是指：对于两个分解序列，若存在$i$使得但是则`$N_1$列必定在$N_2$序列之前输出。每个式子由小到大相加，式子间用分号隔开，且每输出4个式子后换行。

输入样例

7

输出样例

7=1+1+1+1+1+1+1;7=1+1+1+1+1+2;7=1+1+1+1+3;7=1+1+1+2+2
7=1+1+1+4;7=1+1+2+3;7=1+1+5;7=1+2+2+2
7=1+2+4;7=1+3+3;7=1+6;7=2+2+3
7=2+5;7=3+4;7=7

解题思路

这题在PTA当中的标签是递归函数 > 函数与程序结构 > C程序设计。
递归在计算机科学中是指一种通过重复将问题分解为同类的子问题而解决问题的方法。1

要理解递归，必须先理解递归

递归算法解决问题的特点：

  • 在使用递归策略时，必须有一个明确的递归结束条件，称为递归出口
  • 递归算法解题通常比较简洁，但程序运行效率较低
  • 在递归调用的过程当中，系统为每一层的返回点、局部变量等开辟了栈来存储。递归次数过多容易造成栈溢出等，所以一般不提倡用递归算法设计程序。

关于这题，如果要用递归实现，可以考虑从1（加数最小的情况）开始调用函数。当加数之和小于目标时，记录过程并进一步调用；当加数之和大于目标时，直接返回；当加数之和等于目标时，打印结果并返回。
当然，我们需要一个变量来储存中间过程。全局变量执行这一任务比较方便。
难点在于要按题目要求从小到大打印结果，因此，加数的增长应当先从最深的递归函数开始。
