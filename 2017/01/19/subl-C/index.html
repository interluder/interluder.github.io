关于Sublime Text中C编译环境的搭建

发表于 2017-01-19 | 分类于 Tools | 阅读次数

Sublime Text is a sophisticated text editor for code, markup and prose.
You’ll love the slick user interface, extraordinary features and amazing performance.

接触sublime text也不过几个月时间，但与一些传统的IDE（如DevCpp，codeblocks）相比，这款editor确实在编辑代码方面有不俗的表现。这也就是为什么我想利用它来搭建一个可以编译运行C的简单环境，并借此在一定程度上减少对IDE的依赖（然而不得不承认找八阿哥还是IDE好用……也许我这纯属瞎折腾）。当然，如果是写一个比较大的project，还是老老实实用VS吧。

关于编译器

我用的编译器是gcc。
关于gcc的获取，你可以选择直接在从网上下载MinGW（在线版我总是下载失败，也许是被墙了？），或者也可以像我一样偷懒直接用DevCpp里面的gcc（其实也是MinGW），只要找到MinGW64/bin目录下的gcc.exe文件就可以。
MinGW是Minimalist GNU on Windows的首字母缩写，即Windows上的最简GNU。这也就意味着，有了它我们就可以使用GNU的内容软件了，包括gcc、g++、gdb等。

设置环境变量

为了方便，一般我们会把gcc所在的路径加入系统的环境变量，这样就可以直接使用gcc命令。
环境变量可以在系统属性->高级->环境变量中修改。注意：添加路径后用英文分号分隔不同路径。

设置完成后，我们可以通过如下方式检验是否设置成功：
在任意目录打开cmd窗口，输入gcc命令并敲回车运行。如果输出为

gcc: fatal error: no input files
compilation terminated.

那就说明成功了。
如果提示gcc不是内部或外部命令，说明环境变量设置失败。

用cmd编译运行C

这里先给出一些常用的cmd命令。

cd [drive:][path] 显示当前目录名或改变当前目录
dir 显示目录中的文件和子目录列表
md 创建目录
rd 删除一个目录
help 提供 Windows 命令的帮助信息
del/erase 删除一个或数个文件
tree 以图形（树状图）显示驱动器或路径的文件夹结构

至于具体的使用参数，如果忘了或者不知道，可以调用help命令查看。

接下来谈正事。

首先，你可以在你的源文件所在目录下打开cmd窗口——或者你在其他目录打开cmd窗口，然后利用cd命令进入你的源文件所在目录。
然后可以按照gcc <选项> <文件名参数>的语法格式调用gcc命令。

关于选项

-o 指定输出的文件名
-E 仅执行编译预处理
-S 将C代码转换为汇编代码
-Wall 显示警告信息
-c 仅执行编译操作，不执行连接操作

比如，我们要编译helloworld.c这个源文件，并显示警告信息，可以写成如下形式：

gcc -Wall helloworld.c

这里没有指定输出的文件名，在Windows系统中默认输出为a.exe（Linux中为a.out）。

Build System in Sublime Text

事实上，sublime text中有对C++的自带的编译系统。如果之前添加到环境变量的目录中还有g++.exe，那么sublime text中默认的C++编译系统就可以正常工作了。但是这个系统有一个弊端：那就是它只能将程序的输出捕获并显示出来，而不能输入信息（比如执行到含有scanf的语句时）。

解决方案：
如果要编译运行含输入的C，可以调用一个新的cmd窗口，直接在cmd进行输入输出。

那么如何调用cmd呢？这就要用到Sublime Text的Build System了。
在打造自己的Build System之前，不妨先看一下默认的“C++ Single File”是怎么写的。

{
    "shell_cmd": "g++ \"${file}\" -o \"${file_path}/${file_base_name}\"",
    "file_regex": "^(..[^:]*):([0-9]+):?([0-9]+)?:? (.*)$",
    "working_dir": "${file_path}",
    "selector": "source.c, source.c++",

    "variants":
    [
        {
            "name": "Run",
            "shell_cmd": "g++ \"${file}\" -o \"${file_path}/${file_base_name}\" && \"${file_path}/${file_base_name}\""
        }
    ]
}

嗯……看来有必要再花一点点时间了解一下这些JSON格式的配置文件内容究竟代表什么意思了。

cmd： 包含运行指令及其所需参数的数组。如果没有指定绝对路径，它会在你的系统环境变量PATH中搜索所需外部程序。如果shell_cmd未被使用，则需要使用cmd；若shell_cmd已被使用，则被其覆盖（override）。

shell： 可选。若选项为true，cmd将通过shell（cmd.exe, bash…）运行。如果使用了shell_cmd，这个选项将没有作用。

shell_cmd： 具体说明运行指令及其所需参数的字符串。最终需要subprocess.Popen(shell_cmd, shell=True)。（这里不是很懂……好像和python有关系？）当引号的使用较为复杂时，这种写法有助于写出正确的调用。如果cmd未被使用，则需要使用shell_cmd；如果cmd已被使用，则覆盖之。

file_regex： 可选。是Perl格式的正则表达式。可以获取cmd的错误输出。详情参考line_regex

line_regex： 可选。如果file_regex与当前行不匹配，但line_regex存在并且与当前行匹配，则向后遍历缓冲区（buffer），直到与file_regex匹配的行出现，并且用这两个匹配来决定最终要跳转的文件（file）与行（line）。

working_dir： 可选。在运行cmd前会切换到该目录。运行结束后会切换到原来的目录。

selector： 可选。在Tools | Build System | Automatic被勾选时使用。这是一个作用域选择器（scope selector）。Sublime Text用它来为当前视窗找到合适的Build System。

variants： 可选。可用于代替Build System主要选项的备选选项表。如果Build System的选择器与激活的文件匹配，variants下的name就会出现在指令面板（Command Palette）中。

name： 只有在variants中是合法的。（详见variants）用于标识系统中不同的“子Build System”。如果name设为“Run”，则会在Tools | Build System菜单中显示出来。可通过Ctrl + Shift + B来进行调用。

当然，这里我只列举了一部分选项。主要挑选了和上面的内容相关的。其他的也许以后有机会会补上……
如有兴趣可直接查阅Sublime Text非官方文档。

此外，Build System还要用到一些变量。列举如下：

$file_path 当前文件所在路径, 比如 C:\Files
$file 当前文件的完整路径, 比如 C:\Files\Chapter1.txt
$file_name 当前文件的文件名, 比如 Chapter1.txt
$file_extension 当前文件的扩展名, 比如 txt
$file_base_name 当前文件仅包含文件名的部分, 比如 Document
$packages Packages 文件夹的完整路径
$project 当前项目文件的完整路径
$project_path 当前项目文件的路径
$project_name 当前项目文件的名称
$project_extension 当前项目文件的扩展部分
$project_base_name 当前项目仅包括名的部分

下面解释一下

"shell_cmd": "g++ \"${file}\" -o \"${file_path}/${file_base_name}\"",

以‪D:\Cpp\test.cpp为例，这句话相当于在cmd中执行了

g++ "‪D:\Cpp\test.cpp" -o "D:\Cpp\test"

意思就是用g++编译D:\Cpp\test.cpp并在其所在目录下生成test.exe可执行文件。这里用到了\"转义字符，相当于在cmd中输入"。其作用在于，cmd会将引号内部的内容看作一个整体。所以如果你的路径或者文件名当中包含空格，就必须用这种写法。

构建自己的C.sublime-build

事实上，我们只要对上述代码做一些修改。一个简单的版本是这样。

{
    //"encoding": "utf-8",
    "shell_cmd": "gcc -Wall \"$file_name\" -o \"$file_base_name\"",
    "working_dir": "$file_path",
    "file_regex": "^(..[^:]*):([0-9]+):?([0-9]+)?:? (.*)$",
    "selector": "source.c",

    "variants": 
    [
        {    
        "name": "Run in sublime",
            "shell_cmd": "gcc -Wall \"$file_name\" -o \"$file_base_name\" && cmd /c \"${file_path}/${file_base_name}\""
        },
        {    
        "name": "CMD Run",
            "shell_cmd": "gcc -Wall \"$file\" -o \"$file_base_name\" && start cmd /c \"\"${file_path}/${file_base_name}\" & pause\""
        }
    ]
}

这里start cmd /c \"\"${file_path}/${file_base_name}\" & pause\"，就相当于打开新的命令提示符并输入后面相应的命令。
/c： 执行完命令后自动退出。
pause： 暂停，显示请按任意键继续. . .并等待用户按键

但是不建议直接修改系统配置。更好的做法是自己新建一个C.sublime-build。
先找到Tools | Build System | Build New System…，然后输入上述代码，保存为C.sublime-build即可。

关于中文编码乱码的问题

由于Windows命令行中默认的编码格式为GBK，而Sublime Text原生并不支持GBK编码。所以输入中文时常常会遇到乱码的问题。
对于GBK格式的文件，如果安装了ConvertToUTF8插件，Sublime Text就可以正确显示。
如果编译的是UTF-8，而希望在命令行中按GBK的格式正确显示，可以使用编译器选项-fexec-charset来设置代码中字符串的编码。
